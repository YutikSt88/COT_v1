# UI MVP Scope (COT Platform)

## 1) Мета MVP
- Дати трейдеру швидкий доступ до сигналів і позиціонування COT по ринках.
- Забезпечити надійний щоденний сценарій: `скан ринків -> відкриття ринку -> перевірка сигналу`.

## 2) Що входить в MVP

### Екрани
- `Dashboard`
- `Market Detail`
- `Signals`

### Обов'язкові функції
- Вибір ринку (`MarketSelector`)
- Вибір діапазону дат (`DateRangePicker`)
- Відображення поточного сигналу (`SignalBadge`)
- Ключові метрики по ринку (`MetricCard`)
- Графіки позиціонування/екстремумів (`ChartContainer`)
- Табличний перегляд даних (`DataTable`)

### Стан і навігація
- URL-параметри для фільтрів (`market`, `range`, `signal`), щоб можна було ділитися посиланням.
- Єдина темна тема без альтернативних стилів.

## 3) Що НЕ входить в MVP
- Backtest / equity curve / drawdown-аналітика.
- Конструктор стратегій.
- Портфельний модуль.
- Push/Telegram/Email-алерти.
- Складна рольова модель на фронтенді (окрім базового доступу до сторінок).

## 4) Definition of Done (MVP)
- Є 3 робочі екрани: `Dashboard`, `Market Detail`, `Signals`.
- Для кожного екрана працюють фільтри і стан у URL.
- Дані стабільно рендеряться на desktop без UI-критичних помилок.
- Основні таблиці читабельні на dark theme (моноширинні числа, коректне вирівнювання).
- Кольори сигналів послідовні:
  - зелений = bullish/net long
  - червоний = bearish/net short
  - жовтий/помаранчевий = warning/extreme
  - синій = нейтральне/довідкове

## 5) Критерії відсікання (anti-scope creep)
- Якщо задача не покращує сценарій `скан -> деталізація -> рішення`, вона переноситься у `v1/v2`.
- Якщо компонент не перевикористовується мінімум у 2 місцях, він не входить у MVP як окрема сутність.
- Якщо фіча збільшує час реалізації >20% без прямого впливу на якість сигналу, вона поза MVP.
